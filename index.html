What is R2?

R2 is a protocol stack for building applications where your data belongs to you. Devices form trust groups — cryptographic communities where every member can communicate securely without relying on a central server. A browser tab, a microcontroller, a phone, a laptop — all equal peers in the mesh.

Trust Groups, Not Accounts

No usernames, no passwords, no OAuth. Devices join trust groups with cryptographic certificates. The group key encrypts everything. If you're not a member, the data is unreadable.

No Cloud Dependency

The R2 protocol stack runs entirely on your devices. A relay server helps with connectivity (NAT traversal) but never sees your data — it forwards encrypted bytes it cannot read.

Any Transport

BLE, WiFi, LoRa, USB, WebSocket, TCP — R2 treats them all the same. Messages route through whatever path is available. Plug in a USB radio dongle and your laptop is on the mesh.

Browser-Native

The entire R2 stack compiles to 70KB of WebAssembly. Open a web page and your browser becomes a mesh node. No install, no extension, no app store. Just a URL.

Microcontroller to Cloud

The same Rust codebase compiles for bare-metal MCUs (ESP32, RP2040), Linux SBCs (Raspberry Pi), desktop, mobile, and WASM. One protocol, every device class.

Relay Is Infrastructure

Relays route frames by trust group hash. They are untrusted, stateless, and cheap ($5/month VPS or a Raspberry Pi). Multiple trust groups share one relay without seeing each other.

How It Works

R2 applications run in your browser as WebAssembly. The protocol stack handles encryption, trust group management, and mesh routing. A lightweight relay connects devices across the internet.

The Stack

R2 is a seven-layer protocol stack. The Rust implementation compiles to native, WASM, and bare-metal targets from one codebase.

L7 Application GraphQL, edge compute, web UI L6 Sentants Autonomous agents (FSM-based) L5 Trust Ed25519 certificates, HKDF, X25519 join protocol L4 Relay/Dedup TTL, deduplication, relay forwarding L3 Routing Neighbour tables, confidence scoring, transport selection L2 Framing R2-WIRE compact/extended frames, CBOR payloads L1 Transport BLE, WiFi, LoRa, USB, TCP, WebSocket

WASM binary: 182KB (70KB gzipped). Includes full protocol stack with Ed25519, X25519, XChaCha20-Poly1305, HMAC-SHA256.

Hardware

R2 runs on anything — from a $5 microcontroller to a cloud VM. The same Rust codebase compiles for every target.

Tier 1: MCU Node

ESP32-C6, RP2040. Bare-metal Rust. BLE + LoRa + WiFi. Solar-powered, relay-by-default. The R2 dongle — a $32 USB stick that gives any device mesh access.

Tier 2: MCU + SBC

Arduino UNO-Q, Raspberry Pi + LoRa HAT. MCU handles radio (always on), SBC handles compute (wakes on demand). Solar-viable full R2 nodes.

Tier 3: Gateway

Raspberry Pi, always-on Linux. Full R2 stack, bridges mesh to internet. Community charging stations, home gateways, hilltop relays.

Tier 4: Cloud

VPS running the relay. $5/month. Handles NAT traversal for thousands of trust groups. Sees only encrypted frames.

Tier 5: Browser / Mobile

WASM in Chrome/Firefox/Safari. Your browser is a full R2 node. Connect to mesh via WebSocket relay, or directly via WebBluetooth/WebUSB to a dongle.

Mariko EarthGrids

R2's first real-world deployment: community mesh networks for environmental monitoring in Aotearoa New Zealand. Solar-powered LoRa sensors watch rivers and rainfall. Dongles at the marae give everyone mesh access. When infrastructure fails, the mesh keeps working.
